Cuon Client

Hier den Cuon Client für Ubuntu/Debian downloaden. Normalerweise sollte sich jetzt ein entsprechenden Programm zum installieren von .deb Paketen öffnen. Einfach installieren und alles wird gut !
Andernfalls:
Die Datei auf der Festplatte speichern, dann so manuell installieren. Bitte ein Terminal öffnen und folgendes eingeben ( mit Root-Rechten oder als Root)
sudo apt-get install bash wget python-gtk2 python-glade2 python-imaging python-gnome2 python-gtkmozembed python-gtksourceview2
Dann im Downloadverzeichnis dieses hier ausführen:

sudo dpkg -i cuon-client-1.0-all.deb

Jetzt kann jeder Benutzer den Cuon Client vom Menü aus starten.

Hier den Cuon Server für Ubuntu/Debian downloaden. Normalerweise sollte sich jetzt ein entsprechenden Programm zum installieren von .deb Paketen öffnen. Einfach installieren und alles wird gut !
Andernfalls:
Die Datei auf der Festplatte speichern, dann so manuell installieren. Bitte ein Terminal öffnen und folgendes eingeben ( mit Root-Rechten oder als Root)
sudo apt-get install bash, python-reportlab, python-pygre, python-reportlab, postgres, python-pygresql, python-twisted-web, python-twisted-mail, python-twisted-words
Dann im Downloadverzeichnis dieses hier ausführen:

sudo dpkg -i cuon-server-1.0-all.deb

Danach bitte unbedingt dieses Programm aus dem Menü "Einstellungen" den Cuon Server Install starten oder von der Shell aus /usr/bin/cuon_server ausführen, damit bestimmte Pakete, z.B. die Kalenderfunktionen, installiert werden.
Jetzt kann der Administrator den Postgres SQL server konfigurieren und starten, anschließend im Konfigurations client das Setup ausführen.

Danke an Robert für die fantastische Vorarbeit - ohne die wäre ich nie hierangegangen!!!

C.U.O.N. unterscheidet 3 Rechnertypen Server, CuonAdmin, Client Der Server sollte ein Stand-Alone Rechner oder eine virtuelle Maschine jeweils ohne GUI sein. Als einziger Login-User ist root vorgesehen. Hier läuft nur der postgresql im trust-Modus und die cuonserver.
Der CuonAdmin benötigt eine GUI, ebenso der oder die Clients. Es können natürlich 1, 2 oder alle Rechner auf einem Rechner laufen.

Debian Etch

Bitte hier DebianEtch.tar.gz downloaden , entpacken und folgendermaßen vorgehen:
Auf dem Server als Root CuonInstallDebian.sh ausführen, bei Erzeugen des Zertifikats bitte Werte einsetzen oder einfach enter.

Auf dem DuonAdmin als ROOT CuonInstallDebianAdminToolUser.sh ausführen. dann als User CuonInstallDebianAdminToolUser.sh anpassen und ausführen

Auf dem Debian-Linux-Client als Root CuonInstallDebianClient.sh ausführen, dann als User CuonInstallDebianClientUser.sh anpassen und ausführen.
