1.6. Haupt Installation

Jetzt geht es mit den selben Schritten wie bei einem Update weiter.

1.6.1. Server testen und Datenbank einrichten

Zum ersten Testen der Installation langt ein laufender postgresql Server und der cuonxmlrpc Server. Um eine bessere Übersicht über das Geschehen zu haben empfiehlt es sich, den cuonxmlrpc Server per Hand zu starten um die Debug-Ausgabe im Terminal verfolgen zu können: python /usr/share/cuon/cuon_server/src/server_xmlrpc

Dann in einem Fenster Logfile der SQL Datenbank öffnen: tail -f /var/log/postgresql/postg*main*.log cuon starten

Im Linux Startverzeichnis sollten jetzt 3 wichtige cuon Startscripte liegen. Diese bitte so starten:

  • /etc/init.d/cuonxmlrpc start

  • /etc/init.d/cuonweb start

  • /etc/init.d/cuonreport start

Um bei einem Neustart ein automatisches Hochfahren aller benötigten Programme zu gewährleisten kann man mit dem Programm rcconf diese Programmstarts auch schnell einrichten (bitte nicht vergessen die Datenbank postgresql zu starten).
